## 一、加入仓库（第一次必做）

### 第 1 步：注册 GitHub 账号

打开 [https://github.com](https://github.com)，注册一个账号，把你的**用户名**发给管理员。

### 第 2 步：等管理员邀请

管理员会把你加入你专属的仓库（比如 `longmenjihua/longmen-ta-张三`）。

你会收到一封 GitHub 发来的邀请邮件，点击邮件里的 **Accept invitation** 按钮即可。

### 第 3 步：下载仓库到本地

邀请接受后，在终端（Mac）或命令提示符（Windows）运行：

```bash
git clone https://github.com/longmenjihua/longmen-ta-你的名字.git
```

把下载好的文件夹放到你**每天都能找到**的位置，比如桌面。

### 第 4 步：安装自动上传

进入下载好的文件夹：

- **Mac**：双击 `install_mac.sh`，或在终端运行 `bash install_mac.sh`
- **Windows**：右键 `install_windows.bat`，选择「以管理员身份运行」

安装完成后，每天 23:00 会自动把当天内容上传，**不需要你手动操作**。

## 二、Claude 是什么

Claude 是 Anthropic 开发的 AI 助手，你可以把它理解为一个**非常擅长数学的助教搭档**。

**访问地址**：[https://claude.ai](https://claude.ai)（需要注册账号）

龙门计划为 Claude 配置了专属的「技能包」，让它能按照我们的标准格式回答数学题、提炼解题模式，并自动生成所有需要的文件。

## 三、每天怎么用 Claude 答题

### 打开项目

用 Claude Code（命令行工具）打开你的仓库文件夹，Claude 会自动读取 `CLAUDE.md`，加载所有规则和模式库。

```bash
cd 你的仓库文件夹路径
claude
```

### 发题目给 Claude

直接把学生的题目粘贴给 Claude，它会自动：

1. 判断题目难度
2. 查询模式识别库
3. 给出完整解答
4. **自动生成所有文件**（每日输出、每日解答、每日补充、每日反馈）

你不需要手动创建任何文件夹或文件。

## 四、三种使用场景

### 场景 A：学生只发了题目（没有答案）

直接把题目发给 Claude 即可。

Claude 会解题，并自动走完六步工作流程。

**示例输入**：
```
题6. 已知数列 {aₙ} 满足递推关系 aₙ₊₁ = aₙ/(aₙ+1)，a₁ = 1/2，则 a₂₀₂₆ = ?
A. 1/2027  B. 1/2026  C. 1/2025  D. 1/2024
```

---

### 场景 B：学生发了题目 + 答案（让你帮分析思路）

把题目和答案一起发给 Claude，并说明「帮我分析方法」。

Claude 会拆解解题逻辑链，提炼「特征-解法」模式。

**示例输入**：
```
老师，这是题目和答案，帮我分析一下解题思路：

题目：...
答案/解法：...
```

---

### 场景 C：学生想系统学习某个专题

告诉 Claude 你想整理哪个专题的知识体系。

**示例输入**：
```
帮我整理一下数列专题的所有解题方法
```

## 五、自动上传设置（一次性）

> 如果你已经在「加入仓库」第 4 步完成了安装，跳过这一节。

自动上传需要你的电脑上已经登录 GitHub 账号（git 配置）。

**检查是否已登录**：

```bash
git config user.name
git config user.email
```

如果没有输出，运行（替换成你自己的信息）：

```bash
git config --global user.name "张三"
git config --global user.email "zhangsan@example.com"
```

**安装完成后验证**：

手动运行一次上传脚本，看是否成功：

- Mac：`bash auto_push_mac.sh`
- Windows：双击 `auto_push_windows.bat`

查看日志文件 `.longmen/auto_push.log`，出现「上传成功」即正常。

## 六、每天的工作节奏

| 时间 | 你做什么 |
|------|---------|
| 白天 | 用 Claude 回答学生题目，文件自动生成 |
| 每天结束时 | 审查 `每日解答/` 里的 `feedback.md`，填写你的评价 |
| 23:00 | 脚本自动上传，无需操作 |

### 每天结束时需要人工填写的文件

Claude 自动生成的文件中，有几个「空白反馈模板」需要你填写：

- `每日解答/解答[日期]-[时分]/feedback.md`：对 Claude 这道题解答的评价
- `每日补充/补充[日期]-[时分]/补充_[日期时间]_FEEDBACK.md`：对新模式的评价
- `每日反馈/反馈[日期]/feedback[日期].md`：当天整体评价

填完保存即可，23:00 的自动上传会把你的反馈一并提交。

## 常见问题

**Q：Claude 说「库中暂无匹配」怎么办？**

正常现象，Claude 会用其他方法解题，并提炼新模式。你在 feedback 里评价这个新模式是否合理即可。

**Q：忘记填 feedback 怎么办？**

没关系，第二天填也可以，手动运行上传脚本提交。

**Q：上传失败怎么办？**

查看 `.longmen/auto_push.log`，截图发给管理员。

**Q：能不能用网页版 Claude 而不是命令行？**

可以，但网页版无法自动生成文件夹和文件，需要你手动整理。推荐用 Claude Code（命令行）。
