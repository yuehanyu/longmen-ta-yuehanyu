## 一、系统是什么

龙门计划是一套 **AI 辅助答疑系统**。你负责把学生的题目输入给 AI，AI 按照标准格式完成解答，并自动生成所有需要的文件。

你的工作流程：

```
学生发来题目
    ↓
你把题目发给 AI
    ↓
AI 自动完成解答 + 生成所有文件
    ↓
你检查并填写反馈
    ↓
每天 23:00 自动上传
```

**你不需要手动整理任何文件夹，也不需要手动上传。**

## 二、第一次配置（只做一次）

### 第 1 步：注册 GitHub 账号

打开 [https://github.com/signup](https://github.com/signup)，用你的邮箱注册一个账号。

注册完成后，记下你的**用户名**（登录后右上角头像旁边显示的英文名）。

### 第 2 步：Fork 仓库

1. 打开：[https://github.com/longmenjihua/longmen-ta-template](https://github.com/longmenjihua/longmen-ta-template)
2. 点击右上角的 **Fork** 按钮
3. Repository name 改成：`longmen-ta-你的姓名`（例如 `longmen-ta-张三`）
4. 点击 **Create fork**

Fork 完成后，你就有了自己专属的仓库副本。

### 第 3 步：下载到本地

在终端（Mac）或命令提示符（Windows）中运行（替换为你的实际信息）：

```bash
git clone https://github.com/你的GitHub用户名/longmen-ta-你的姓名.git
```

### 第 4 步：运行 Skill 完成剩余配置

进入下载好的文件夹，打开 Claude Code（AI 命令行工具），加载 `ta-setup.skill`，说：

```
帮我安装
```

AI 会引导你完成：填写姓名、安装自动上传脚本、开启自动同步。

### 第 5 步：发送入组申请

Skill 运行结束时会生成一段消息，**复制后发给管理员**，格式如下：

```
【龙门计划 - 新助教入组申请】
姓名：张三
GitHub 用户名：zhangsan123
仓库地址：https://github.com/zhangsan123/longmen-ta-张三
```

管理员收到后会把你的仓库接入汇总系统，你就正式加入了。

## 三、每天怎么工作

### 开始工作

打开 Claude Code，进入你的仓库文件夹。AI 会自动读取系统规则，准备好接收题目。

### 处理题目

把学生的题目直接发给 AI，它会自动：

1. 查询模式识别库，匹配已有解法
2. 完成解答或模式分析
3. 生成以下文件：
   - `每日输出/output[日期].md` — 当天所有问答汇总
   - `每日解答/解答[日期]-[时分]/解答.md` — 单题人话版讲解
   - `每日解答/解答[日期]-[时分]/feedback.md` — 待你填写的反馈表
   - `每日反馈/反馈[日期]/feedback[日期].md` — 当天整体反馈表
   - `每日补充/补充[日期]-[时分]/...` — 发现新模式时自动创建

### 填写反馈（每天结束时）

AI 生成的 `feedback.md` 是空白模板，需要你填写评价。打开这些文件，填写你对 AI 解答质量的意见。

填完保存即可，23:00 的自动上传会一并提交。

### 自动上传

每天 **23:00**，脚本自动将当天所有内容上传到 GitHub，**不需要你做任何操作**。

## 四、三种答题场景

### 场景 A：学生只发题目（没有答案）

直接把题目发给 AI，它负责解题。

```
示例输入：
已知数列 {aₙ} 满足 aₙ₊₁ = aₙ/(aₙ+1)，a₁ = 1/2，则 a₂₀₂₆ = ？
```

### 场景 B：学生发了题目 + 答案（要求分析思路）

把题目和答案一起发给 AI，说明「帮我分析方法」。

```
示例输入：
这是题目和答案，帮我分析一下解题思路：
题目：……
答案/解法：……
```

AI 会拆解逻辑链，提炼「特征 → 解法」模式。

### 场景 C：系统整理某个专题

```
示例输入：
帮我整理一下数列专题的所有解题方法
```

## 五、文件在哪里看

你的仓库文件夹结构：

```
longmen-ta-张三/
├── 模式识别库/          ← AI 查询用的解题模式（自动同步，不要手动修改）
├── Skill/              ← AI 技能包（自动同步，不要手动修改）
├── CLAUDE.md           ← AI 的工作规则（自动同步，不要手动修改）
│
├── 每日输出/            ← 每天所有问答汇总
├── 每日解答/            ← 每题的详细解答 + 反馈表（你需要填写 feedback.md）
├── 每日补充/            ← 新模式记录 + 反馈表（你需要填写反馈）
└── 每日反馈/            ← 当天整体评价（你需要填写）
```

前三项（模式识别库、Skill、CLAUDE.md）由管理员统一维护，你的仓库会自动同步，**不需要你手动操作**。

## 六、内容自动更新说明

管理员会不定期更新模式库、技能包、系统规则等内容。

你的仓库每天早上 **8:00** 自动从母版同步这些更新，**完全不需要你操作**。

如果管理员通知你「有紧急更新，请立刻同步」，在 Claude Code 里说：

```
同步
```

AI 会立刻执行同步，几秒内完成。

> 注意：同步只会更新系统文件，你的每日解答、每日补充、每日反馈等工作文件**永远不会被覆盖**。

## 七、常见问题

**Q：AI 说「库中暂无匹配」是出问题了吗？**

不是。这表示这道题的解法模式还没录入库，AI 会用其他方法解题并提炼新模式。你在 feedback 里评价这个新模式是否合理即可。

**Q：自动上传失败怎么办？**

查看 `.longmen/auto_push.log` 文件，截图发给管理员。

**Q：能不能手动上传？**

可以：
- Mac：在终端运行 `bash auto_push_mac.sh`
- Windows：双击 `auto_push_windows.bat`

**Q：忘记填 feedback 了怎么办？**

没关系，补填后手动上传一次即可。

**Q：Claude Code 怎么安装？**

请参阅管理员提供的《API 接入文档》。
